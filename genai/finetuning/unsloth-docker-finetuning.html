Unsloth aur Docker se LLM Fine-tune kaise karein üê≥

Local training aksar dependency issues ya environment break hone ki vajah se complex ho sakti hai. Unsloth ki Docker image in sabhi problems ko bypass kar sakti hai. Koi setup zaroori nahi hai: bas image ko pull karein, run karein aur training shuru karein.

  • Unsloth official Docker image: `unsloth/unsloth`

Unsloth aur Docker kyun use karein?

Unsloth ki Docker image stable, up-to-date hai aur Windows jaise supported setups mein kaam karti hai.

  • Fully contained dependencies aapke system ko clean rakhti hain. Bina root ke safely chalti hai.
  • Ise locally ya kisi bhi platform par pre-installed notebooks ke saath use karein.
Aap ab hamari main Docker image `unsloth/unsloth` ko Blackwell aur 50-series GPUs ke liye use kar sakte hain - alag se koi image zaroori nahi hai.

‚ö° Step-by-Step Tutorial

Docker aur NVIDIA Container Toolkit install karein.

Docker ko Linux ya Desktop (other) ke zariye install karein. Phir NVIDIA Container Toolkit install karein:

export NVIDIA_CONTAINER_TOOLKIT_VERSION=1.17.8-1
sudo apt-get update && sudo apt-get install -y \
  nvidia-container-toolkit=${NVIDIA_CONTAINER_TOOLKIT_VERSION} \
  nvidia-container-toolkit-base=${NVIDIA_CONTAINER_TOOLKIT_VERSION} \
  libnvidia-container-tools=${NVIDIA_CONTAINER_TOOLKIT_VERSION} \
  libnvidia-container1=${NVIDIA_CONTAINER_TOOLKIT_VERSION}

Container run karein.

`unsloth/unsloth` Unsloth ki eklauti Docker image hai. Blackwell aur 50-series GPUs ke liye, yahi image use karein - alag se koi image zaroori nahi hai. Agar DGX Spark use kar rahe hain, to aapko hamari DGX guide follow karni hogi.

docker run -d -e JUPYTER_PASSWORD="mypassword" \
  -p 8888:8888 -p 2222:22 \
  -v $(pwd)/work:/workspace/work \
  --gpus all \
  unsloth/unsloth

Jupyter Lab access karein.

http://localhost:8888 par jaakar Unsloth open karein. `unsloth-notebooks` tab ko access karke Unsloth notebooks dekhein.

Unsloth ke saath training shuru karein.

Agar aap naye hain, to hamari step-by-step Fine-tuning Guide, RL Guide follow karein ya hamare premade notebooks mein se kisi ko bhi save/copy karein.

docker run -d -e JUPYTER_PORT=8000 \

  -e "SSH_KEY=$(cat ~/.ssh/container_key.pub)" \
  -e USER_PASSWORD="unsloth2024" \
  -p 8000:8000 -p 2222:22 \
  -v $(pwd)/work:/workspace/work \
  --gpus all \
  unsloth/unsloth

SSH Key set up karna

Agar aapke paas SSH key pair nahi hai:

# Naya key pair generate karein
ssh-keygen -t rsa -b 4096 -f ~/.ssh/container_key

# Public key ko docker run mein use karein
-e "SSH_KEY=$(cat ~/.ssh/container_key.pub)"

# SSH se connect karein
ssh -i ~/.ssh/container_key -p 2222 unsloth@localhost

üîí Security Notes

  • Container default roop se non-root `unsloth` user ke roop mein chalta hai.
  • Container ke andar sudo operations ke liye `USER_PASSWORD` ka upyog karein.
  • SSH access ke liye public key authentication ki zaroorat hoti hai.
